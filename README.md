## 配裝計算器使用方式

### 1. 匯入資料

- **角色 CSV**：點擊「角色 CSV」按鈕，匯入 `chars.csv`（或從 GitHub 自動預載）
- **裝備 JSON**：從遊戲導出裝備資料後，點擊「裝備 JSON」按鈕匯入
  - 匯入後資料會快取在瀏覽器 localStorage，下次開啟不需重新匯入
  - 右上角顯示 💾 icon 表示有快取，hover 可看快取時間

### 2. 配裝設定（Mode A / Mode B）

**Mode A — 自由配置（推薦新手）**
- 設定各屬性的「下限門檻」和「優化權重（0~1）」
- 系統依加權分數排序，適合平衡型配裝

**Mode B — 目標鎖速**
- 先設速度區間，再選擇要最大化的單一屬性
- 適合速度已確定、只想最大化特定屬性的情境

### 3. 主屬鎖定

在套裝需求旁可鎖定項鍊/戒指的主屬，避免被非目標主屬裝備佔滿候選池：
- 💎 項鍊：爆率 / 爆傷 / 攻擊% / 生命% / 防禦% / 攻擊 / 生命 / 防禦
- 💍 戒指：命中 / 抗性 / 防禦% / 生命% / 攻擊% / 防禦 / 生命 / 攻擊

### 4. 超級鎖定 🔐

在裝備庫中點擊 🔐 按鈕，可以將裝備設為超級鎖定：
- 超級鎖定的裝備在所有角色的配裝中一律排除
- 適合確定要給某個角色專用的核心裝備
- 一般鎖定（🔒）只排除其他角色，超級鎖定完全排除

### 5. 鎖定方案

計算出結果後，點擊「儲存方案」可存入 1～3 號方案：
- 進入「🔒 鎖定/手動」tab 查看已儲存的方案
- 每個角色卡內有 1號/2號/3號/手動 四個分頁

### 6. 手動配套（🖐 手動）

在鎖定 tab 的「🖐 手動」分頁，可以手動選擇 6 件裝備：
- 左三：武器、頭盔、鎧甲；右三：項鍊、戒指、鞋子
- 即時顯示與已儲存方案的數值差異（綠=提升 / 紅=下降）
- 可帶入已有的計算器方案，再手動微調

## 資料檔案維護

### chars.csv — 角色資料

**格式：**
```
version:YYYYMMDD
角色本名,屬性,職業,base_atk,base_def,base_hp,base_spd,bonus_atk,bonus_def,bonus_hp,bonus_spd,innate_cr,innate_cd
```

**維護注意事項：**
- 第一行必須是 `version:YYYYMMDD`，版本日期改變才會觸發使用者重新載入
- 角色本名要和 build_recom.csv 完全一致（含空格、全形字元）
- 被動技能加成（如固有暴擊率）填入 `innate_cr`，不要加進 base 屬性

---

### build_recom.csv — 推薦配裝

**格式（第一行為版本行）：**
```
version:YYYYMMDD
角色名,方案編號,方案名稱,方案說明,套裝1,套裝2,套裝3,速度下限,速度上限,爆率下限,爆率上限,爆傷下限,攻擊下限,防禦下限,生命下限,命中下限,抗性下限,權重_攻擊,權重_爆率,權重_爆傷,權重_命中,權重_抗性,權重_生命,權重_防禦,速度鞋,TAG
```

#### 新增/修改推薦配裝步驟

1. **確認角色名稱**  
   在 `chars.csv` 找到「角色本名」欄位，**複製貼上**，不要手動輸入（避免空白/全形差異）

2. **決定方案數量**  
   同一角色可以有 1~3 個方案，方案編號從 1 開始

3. **填寫門檻與權重**  
   - 門檻（下限）：最低達標要求，不達標的組合直接排除  
   - 權重：0~1，值越高排序越優先；建議步進 0.1  
   - 空白欄位 = 不限制/不考慮

4. **更新版本日期**  
   修改第一行 `version:YYYYMMDD` 為今天日期，否則使用者讀到的還是舊快取

5. **上傳到 GitHub**  
   直接在 GitHub 網頁編輯，或 commit push

#### 欄位說明

| 欄位 | 說明 | 範例 |
|---|---|---|
| 角色名 | 必須與 chars.csv 完全一致 | 繼任者彌卡伊勒 |
| 方案編號 | 同角色多方案時的排序 | 1 |
| 方案名稱 | Popup 標籤文字，簡短 | 極速坦克 |
| 方案說明 | 一句話說明核心思路 | 速度 180+ 後全力堆生命 |
| 套裝1~3 | 指定套裝名稱，空白=不限 | 速度 |
| 速度下限/上限 | 整數，空白=不限 | 180 |
| 爆率/爆傷下限 | % 整數（85 代表 85%），空白=不限 | 85 |
| 攻擊/防禦/生命下限 | 固定數值整數，空白=不限 | 20000 |
| 命中/抗性下限 | % 整數，空白=不限 | 25 |
| 權重_各屬性 | 0~1 小數，0=不考慮此屬性 | 0.8 |
| 速度鞋 | 1=建議速度主屬鞋；空白=不限 | 1 |
| TAG | 分號分隔，見下方 TAG 清單 | 速度型;解控型 |

#### 有效 TAG 清單

| TAG | 適用場景 |
|---|---|
| 速度型 | 速度優先的一速角色 |
| 輸出型(高速) | 高速+爆率爆傷的攻擊角色 |
| 輸出型(低速) | 低速但輸出為主 |
| 坦克型 | 生命/防禦為主的坦角 |
| 肉輸出 | 攻防兼備 |
| 控制型 | 控制技能為主 |
| 解控型 | 持有群體解控技能 |
| 拉條型 | 可拉升己方行動條 |
| 絕對一速 | 必須全隊最快 |
| 對策額外回合 | 對付擁有額外回合的敵人 |
| 對策非攻擊技能 | 對付非攻擊技能觸發的敵人 |

#### 常見錯誤

- ❌ **角色名不一致**：CSV 名稱和 chars.csv 不完全相符時，Popup 靜默失敗（不顯示，無錯誤提示）
- ❌ **% 值填小數**：爆率/爆傷下限應填整數（85），不是小數（0.85）
- ❌ **版本日期沒更新**：使用者快取不會更新，仍看到舊資料
- ❌ **方案編號重複**：同一角色方案編號重複時，後面的覆蓋前面的

```csv
version:20260314
望月,1,一速啟動,速度 245+ 確保先手，搭配魂書燒魂,速度,,,245,,85,,200,,,,,,0.4,0.1,0.3,0,0,0,0,1,絕對一速
艾瑞兒,1,極速坦克,速度優先吸收傷害保護隊伍,復仇,,,180,,,,,,,20000,,,0,0,0,0,0,0.7,0.4,1,肉輸出;坦克型
艾瑞兒,2,純坦肉盾,不需速度時的極致坦度方案,生命,,,,,,,,,1400,26000,,,0,0,0,0,0,1,0.6,0,坦克型
```

## 技術架構

- **前端框架**：React 18（UMD CDN）+ Babel Standalone 7.23.10
- **樣式**：Tailwind CSS CDN
- **資料儲存**：瀏覽器 localStorage（無後端）
- **部署**：GitHub Pages（推送即部署）

## 開發注意事項

- JSX 中的 template literal（反引號字串）需改為字串拼接，`className={}` 除外
- `import` 語句不可使用，必須用 `const { useState, ... } = React`
- `export default function App()` 必須改為 `function App()`
- 詳見開發規格文件 `arkrecode_dev_spec_v2.5.0.docx`
