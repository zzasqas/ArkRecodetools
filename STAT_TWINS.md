# stat-twins — 角色數值模板對照表

## 這是什麼

`chars.csv` 裡不少角色的數值其實是共用同一套「模板」——不同角色、不同屬性、不同外觀，
但基礎攻/防/血/速完全一樣，只是調教（練度）點法不同而已。

`stat-twins.html` 就是把這些「數值雙胞胎（甚至多胞胎）」抓出來列成表，方便：

- 幫新角色抓數值時，找一個同模板的舊角色直接對照，減少抄錯的機會
- 練度規劃時知道「這個角色跟那個角色骨架一樣，養法可以參考」
- 單純好奇官方到底是重複用了多少套底子

## 怎麼看

雙擊開啟 `stat-twins.html`（純靜態頁面，不需要跑伺服器）：

- **強化前 / 強化後** 切換：
  - 「強化前」＝基礎值（沒有調教加成前的攻防血速）分組——這是判斷「是不是同一套模板」最準的依據。
  - 「強化後」＝總值（基礎值＋調教加成）分組——同一個模板的角色如果調教點法不同，總值可能就會被拆成好幾個小群組。
  - 兩個切換之間的差異，就是「調教點數把原本一樣的角色拉開多少」。
- 星級 / 職業篩選：縮小範圍用。
- 每列的「模板名稱」是人工取的花名（例如「低血高防術師模板」「脆皮爆發先鋒模板」），
  方便記憶跟討論；沒有花名的新群組會自動顯示「{職業}共用模板」。

## 資料怎麼來的、怎麼更新

由 `scripts/find_stat_twins.py` 讀根目錄 `chars.csv` 產生，不含三星角色（雜訊太多）。
新角色加入 `chars.csv` 後，重跑一次就會更新：

```powershell
python scripts/find_stat_twins.py
```

## 幫新模板取花名

如果新角色湊出一組新的「共用模板」，腳本會印出提醒：

```
提醒：有 N 組沒有花名，用了 fallback 通用名稱，可手動補進 TEMPLATE_LABELS
```

打開 `scripts/find_stat_twins.py`，在檔案開頭的 `TEMPLATE_LABELS` 字典裡加一行即可：

```python
(5, (基礎攻, 基礎防, 基礎血, 基礎速)): "取個花名",
```

key 是 `(星級, 基礎值 tuple)`——花名綁在「基礎值」上，這樣即使角色在「強化後」視角被
練度拆開到不同群組，也會沿用同一個花名，看得出他們本來是同一套模板。

## 已知限制

- 不含三星角色。
- 花名是人工判斷貼的標籤，不是演算法生成，新群組出現時需要手動補（見上）。
- 只比對攻/防/血/速四項基礎/總值，不比較爆擊率、爆傷、狀態命中/抗性、被動類型等欄位。
